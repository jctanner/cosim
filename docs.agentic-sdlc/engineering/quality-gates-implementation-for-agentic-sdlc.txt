# Quality Gates Implementation for Agentic SDLC

## Overview
This document defines the concrete implementation of quality gates for agent-generated code, derived from the RHOAI baseline workflow. The baseline uses 4 manual review checkpoints; we're replacing these with automated gates plus strategic human triggers.

## 1. Pre-Commit Static Analysis Gates

### Configuration (`.pre-commit-config.yaml`)
```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.3.0
    hooks:
      - id: ruff
        args: [--fix, --exit-non-zero-on-fix]
      - id: ruff-format
  
  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.8.0
    hooks:
      - id: mypy
        args: [--strict, --ignore-missing-imports]
  
  - repo: https://github.com/PyCQA/bandit
    rev: 1.7.7
    hooks:
      - id: bandit
        args: [-ll, -r, ., --severity-level=medium]

  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: check-added-large-files
        args: [--maxkb=500]
      - id: check-merge-conflict
      - id: check-yaml
      - id: detect-private-key
```

### Auto-Reject Thresholds
- **Complexity:** Cyclomatic complexity > 15 (auto-reject)
- **Security:** Any medium+ severity Bandit findings (auto-reject)
- **Type safety:** MyPy strict mode violations (auto-reject)
- **Code style:** Ruff violations (auto-fix, then reject if still failing)

## 3. Mutation Testing Automation

### Configuration (`mutmut.toml`)
```toml
[mutmut]
paths_to_mutate = "src/"
tests_dir = "tests/"
runner = "pytest -x --tb=short"
minimum_score = 85.0  # Auto-reject if mutation score < 85%
```

### CI Integration
```bash
#!/bin/bash
# Run mutation testing on agent-generated code
mutmut run --paths-to-mutate src/
SCORE=$(mutmut results | grep "Mutation score" | awk '{print $3}' | sed 's/%//')

if (( $(echo "$SCORE < 85.0" | bc -l) )); then
    echo "FAIL: Mutation score $SCORE% below threshold 85%"
    exit 1
fi

echo "PASS: Mutation score $SCORE%"
```

## 4. Human Approval Triggers

### Approval Automation Rules (`approval_rules.yaml`)
```yaml
rules:
  - name: "New External Dependencies"
    trigger:
      file_pattern: "requirements.txt|package.json|go.mod"
      change_type: "addition"
    action: "require_human_approval"
    approvers: ["security_team", "architect"]
  
  - name: "Database Schema Changes"
    trigger:
      file_pattern: "**/migrations/**|**/schema.sql"
    action: "require_human_approval"
    approvers: ["dba", "architect"]
  
  - name: "Authentication/Authorization Logic"
    trigger:
      code_pattern: "(authenticate|authorize|permission|oauth|jwt)"
      file_pattern: "**/auth/**|**/security/**"
    action: "require_human_approval"
    approvers: ["security_team", "senior_engineer"]
  
  - name: "Customer-Facing API Changes"
    trigger:
      file_pattern: "**/api/v[0-9]+/**"
      change_type: "breaking"
    action: "require_human_approval"
    approvers: ["architect", "product_manager"]
```

### Git Commit Hook
```python
#!/usr/bin/env python3
# .git/hooks/prepare-commit-msg
import sys
import json
import os

def inject_agent_metadata(commit_msg_file):
    metadata = {
        "agent_id": os.getenv("AGENT_ID", "unknown"),
        "execution_timestamp": datetime.utcnow().isoformat(),
        "quality_metrics": get_quality_metrics(),
    }
    
    with open(commit_msg_file, 'a') as f:
        f.write(f"\n\nAgent-Metadata: {json.dumps(metadata)}")

if __name__ == "__main__":
    inject_agent_metadata(sys.argv[1])
```
